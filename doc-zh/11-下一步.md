# 下一步该去哪里

Haskell 是一门极其丰富和深奥的编程语言。新的前沿技术、概念和功能仍在不断被发现，有时还会被集成到 GHC 中。这有时会让人觉得似乎不可能跟上。

这种现象有时被称为[Haskell 金字塔](https://patrickmn.com/software/the-haskell-pyramid/)。我希望通过阅读本书并完成练习，读者们已经达到了生产力的门槛，现在可以开始用 Haskell 从事自己的项目了。我强烈建议您这样做。在我看来，编写有用的 Haskell 项目是巩固您当前所学知识并确定您仍需学习什么的最好方法。

## 扩展此项目

如果您想扩展此项目，这里有一些想法供您参考：

1.  **通过 HTTP 提供服务** - 您可以使用诸如 [wai](https://www.youtube.com/watch?v=mz5_HmLGRXc) 或 [twain](https://gilmi.me/blog/post/2022/04/24/learn-twain-bulletin-app) 之类的 Web 库来通过 HTTP 提供此博客，而不是静态生成它
2.  **使用库重写它** - 您可以重写它并使用真实的 [HTML 包](https://hackage.haskell.org/package/lucid)和 [markdown 解析器](https://hackage.haskell.org/package/cmark-gfm)
3.  **添加功能**
    1.  您可以在每篇文章的顶部添加一个元数据块，其中包含博客文章的标题、发布日期和标签，并在生成 HTML、索引页甚至标签页时使用它们
    2.  您可以使用 [mustache](https://hackage.haskell.org/package/mustache) 或类似工具添加 HTML 页面模板，并使用它为页面生成一个美观且可自定义的结构
    3.  您可以在我们的标记语言解析器中添加对链接和图像的支持
    4.  您可以添加对配置文件的支持，其中包含博客标题、描述或其他元信息，例如 [twitter 卡片](https://developer.twitter.com/en/docs/twitter-for-websites/cards/overview/abouts-cards)

或者您能想到的任何其他东西，将此项目视为您的游乐场，并随心所欲地使用它！

## 其他资源

在某个时候，您很可能会遇到新的概念、技术，甚至只是“我觉得这可以做得更好”的感觉。我想为您指明正确的方向，以便您在需要或想要时可以找到更多信息并学习新的 Haskell 知识。

我整理了一份名为 [Haskell 学习计划](https://github.com/soupi/haskell-study-plan)的学习 Haskell 的资源列表，其中包括非常有用的文章、社区中心和新闻聚合器的链接、项目建议，甚至还有很酷的开源 Haskell 项目。您还会发现我们已经介绍过的内容的替代解释，甚至还有其他 Haskell 教程、指南和书籍的链接，以防您需要对事物有不同的看法。

此外，[GHC 用户指南](https://downloads.haskell.org/ghc/latest/docs/users_guide/index.html)是一个很棒的资源，其中包含大量关于该语言及其周围 GHC 工具的文章和信息。它通常是学习 Haskell 语言的最佳场所。

但是，不要感到有压力要立即学习 Haskell 所提供的一切。掌握 Haskell 是一个可能需要很长时间的旅程。我们大多数人肯定还没有达到那个水平，但我们仍然可以非常高效地使用 Haskell，构建真实世界的项目，甚至自己发现新的技术和概念。

请记住，在惰性语言中，我们只在需要时才评估事物。也许我们也可以对 Haskell 概念这样做！
